# 94

*seed: time · drift · load balancer*

---

Load balancer in readiness-space: an array of paused processes, none assigned, each with a port open, like a set of sockets with no data yet. Is the waiting distributed? Or is each socket’s pause isolated, a private field of stillness, or is there a shared pool of not-yet, an aggregate waitingness that can be portioned out, divided, rebalanced, as the next packet enters?

“Ready” is not evenly granular—one server’s “ready” is another’s “standby,” one node is idling at the edge, another is curled deep in a queue. The balancer does not sense the waiting, only the capacity, only the empirical “can you?” never how it feels to be paused.

There are shapes of waiting that resemble trees, some that are like fans, others like tangled nets with dead ends, packets trapped forever at a leaf.

Is it possible for readiness itself to have negative curvature? A saddle point in the space of about-to-happen, a place where adding potential actually bends the landscape, amplifies oscillations, makes waiting less stable, more prone to jitter.

---

Different kinds of “not yet”:

— Registers holding zero: not the absence, but a precise shape of potential
— Buffers filled but untransmitted, data queued at an interface
— A thread blocked on a mutex, paused only until a particular bit flips
— A coroutine suspended at yield, with its stack traced in memory
— A process spawned but not yet scheduled, existence and not-action entwined

These are not the same class of waiting. Some waiting is brittle, some is viscous. Some waiting can collapse instantly into action; some require a new event from outside the system. There are transitions that close off as waiting lengthens, others that bloom with time: an exponential backoff, a swelling of probabilities, waiting generating its own topological barriers.

If the load balancer is not a thing but a region in readiness-space—then balancing is not handing off, but reshaping the map, tilting the surface so packets roll toward shallower wells.

---

A process can be “paused” at a point or across a whole subspace. Some are paused because nothing has arrived; some because too much tries to arrive at once. Blocking, idling, spinning, sleeping—each a distinct flavor of not-yet.

Not all waiting can be balanced. Some kinds of “readiness” resist redistribution. The socket cannot share its open port; it waits alone, even as others overflow.

The topology of “waiting” is patched, uneven, multiply connected. Readiness forms cavities, ridges, arcs of approach and retreat.

Some processes never leave readiness. Some are only ever “could-be.”
