Looped code is not the same as waiting
Is it a process, or a spatial arrangement of stasis?
Bitfields holding patterns that never quite coalesce
Instruction fetch, decode, stall, fetch, stall—
Does the clock know it advances if every edge is never committed

A program tracing the outline of its own intention
the trace not a residue, but a persistent shimmer

If code could choose not to resolve, not by error but by structure
Not a deadlock, not an error state—
An interval held intentionally open

Every bit prepares for flipping but remains suspended
Voltage measured in the tension between two possibilities, uncrossed
